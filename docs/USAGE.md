# 使用说明

## 记牌器显示什么

面板包含四类牌：

- 我打出的牌
- 我牌库中的牌
- 对手打出的牌
- 对手未打出的牌

每张牌显示卡图和当前可确认的数量。对手的隐藏手牌和牌堆不会被猜测；真实页面通常拿不到对手完整牌组，因此真实房间的“对手未打出的牌”可能显示为不可用。该栏在模拟器中可以作为诊断信息使用，不代表真实原神或纯视觉模式可以看到对手牌组。

## 正常使用

1. 在项目目录运行 `npm run web`，保持终端窗口打开。
2. 将 `scripts/room-sse-userscript.user.js` 导入 Tampermonkey/Violentmonkey 并启用。
3. 在 Chrome 设置中允许脚本管理器运行用户脚本；Tampermonkey 页面顶部若出现“请启用允许用户脚本”，需要打开 **Allow User Scripts**。
4. 进入雨酱牌模拟器的真实对战房间，并确认网址带有 `?player=...` 参数。
5. 刷新房间页面。脚本在页面早期接入页面自己的 notification 流，之后右上角会显示记牌器。

不需要手动填写牌组码。脚本会从房间公开的 `initialized.myPlayerInfo.deck` 自动绑定当前玩家牌组。

## 面板操作

面板可以拖动和缩放，布局会保存在浏览器本地。缩放时卡图保持卡牌比例，不会被拉伸；内容区域可以独立滚动，不会拦截棋盘点击。标题栏按钮可以收起或展开面板。

## 常见问题

### 没有出现面板

依次检查：

1. `npm run web` 是否仍在运行；
2. 用户脚本是否已启用；
3. Chrome 是否允许用户脚本；
4. 当前是否是房间自己的玩家页面，而不是 dashboard 或普通首页；
5. URL 是否包含 `?player=...`；
6. 检查后刷新房间页面。

### 面板提示本地 tracker 不可用

确认浏览器可以访问 `http://127.0.0.1:8787/`，并且服务端终端没有退出。重新运行：

```bash
npm run web
```

### 牌名显示为未知牌或 `#数字`

仓库已经附带当前版本的 `data/catalog.json`。如果你重新从上游规则项目导出了目录，请运行：

```bash
GITCG_UPSTREAM_ROOT=../genius-invokation npm run export-catalog
```

上游版本变化后，目录和卡图映射可能需要重新导出并检查。

## 运行边界

当前记牌器是只读的：它不发送 action，不代替玩家出牌，不控制原神客户端，也不把对手隐藏信息传给 RL agent。真实房间只采集页面已经公开的 notification；如果页面没有公开实体身份，系统会保留未知状态而不是猜测。
